# 🧵 Практична задача: Асистент етно-майстерні

## 📌 Огляд
Це завдання розроблене для закріплення знань з **Розділів 5 та 6** курсу C#. Ти виступиш у ролі розробника системи для майстра, який поєднує старовинні традиції та сучасний побут.

## 🛠 Технічні вимоги

### 1. Класифікатор виробу (`if-else` + Логічні оператори)
Реалізуй перевірку типу виробу та матеріалу, щоб зрозуміти його автентичність.
- **Вхідні дані:** `category` (рядок), `material` (рядок).
- **Логіка:**
    - Якщо **категорія** "Одяг" **ТА** **матеріал** "Льон" — вивести: *"Це автентична сорочка (старовинний стиль)"*.
    - Якщо **категорія** "Одяг" **ТА** **матеріал** НЕ "Льон" — вивести: *"Це сучасний текстильний виріб"*.
    - Якщо **категорія** "Посуд" **ТА** **матеріал** "Глина" — вивести: *"Це кераміка ручної роботи"*.
    - В іншому випадку — вивести: *"Категорія для хендмейду не визначена"*.

### 2. Вибір техніки декору (`switch expression`)
Використай сучасний синтаксис `switch` для автоматичного вибору техніки обробки.
- **Вхідні дані:** `region` (рядок).
- **Результати (через switch expression):**
    - `"Полтава"` => "Вишивка біллю (білим по білому)"
    - `"Гуцульщина"` => "Низинка або кучерявий шов"
    - `"Сучасний"` => "Машинна вишивка або принт"
    - `_` (дефолт) => "Техніка обирається майстром"

### 3. Перевірка терміновості (Тернарний оператор)
Визнач, чи потрібно майстру працювати вночі.
- **Вхідні дані:** `daysToDeadline` (ціле число).
- **Логіка:** Якщо днів до здачі менше **3**, встановити статус "Терміново", інакше — "Звичайно".
- **Результат:** Вивести у консоль: *"Статус замовлення: [Результат]"*.

## 💡 Поради від Ментора

> [!TIP]
> Для порівняння рядків використовуй метод `.ToLower()`, щоб програма була стійкою до регістру (наприклад, щоб "Льон" і "льон" сприймалися однаково).

### Приклад синтаксису Switch Expression:
```csharp
string result = region switch
{
    "Полтава" => "Вишивка біллю",
    "Гуцульщина" => "Низинка",
    _ => "Стандарт"
};
