# 🧵 Практична задача: Асистент етно-майстерні (v2.0)

## 📌 Огляд
Ти створюєш інтерактивний консольний додаток для майстрів хендмейду. Програма повинна вести діалог з користувачем, збирати дані та видавати персоналізовані поради.

---

## 🎮 Дії користувача (User Flow)

Щоб програма працювала коректно, реалізуй наступну послідовність дій:

1.  **Привітання:** Програма виводить повідомлення: *"Вітаємо в асистенті майстерні! Давайте оформимо ваше замовлення."*
2.  **Введення категорії:** Користувач вводить тип виробу (Одяг/Посуд/Інше).
3.  **Введення матеріалу:** Користувач вводить назву матеріалу (Льон/Глина/Пластик тощо).
4.  **Вибір локації:** Програма запитує: *"Для якого регіону виконується робота?"* (Полтава/Гуцульщина/Сучасний).
5.  **Встановлення дедлайну:** Користувач вводить кількість днів, що залишилися до здачі роботи.
6.  **Отримання результату:** Програма миттєво обробляє дані та виводить підсумок.

---

## 🛠 Технічні вимоги

### 1. Класифікатор виробу (`if-else` + Логічні оператори)
- Якщо **категорія** "Одяг" **ТА** **матеріал** "Льон" — вивести: *"Це автентична сорочка (старовинний стиль)"*.
- Якщо **категорія** "Одяг" **ТА** **матеріал** НЕ "Льон" — вивести: *"Це сучасний текстильний виріб"*.
- Якщо **категорія** "Посуд" **ТА** **матеріал** "Глина" — вивести: *"Це кераміка ручної роботи"*.
- В іншому випадку — вивести: *"Категорія для хендмейду не визначена"*.

### 2. Вибір техніки декору (`switch expression`)
Визнач техніку залежно від регіону:
- `"Полтава"` => "Вишивка біллю (білим по білому)"
- `"Гуцульщина"` => "Низинка або кучерявий шов"
- `"Сучасний"` => "Машинна вишивка або принт"
- `_` => "Техніка обирається майстром"

### 3. Перевірка терміновості (Тернарний оператор)
- Якщо днів до здачі менше **3**, встановити статус "Терміново", інакше — "Звичайно".
- Вивести: *"Статус замовлення: [Результат]"*.
